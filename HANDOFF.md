## How it's built
Everything is generated by **`tools/build.py`** from `content/raw/*.json` + a set of override CSVs. Run `python3 tools/build.py` to regenerate. Theme = dark charcoal + orange (`#E8720C`), Montserrat/Inter, in `assets/site.css` + `assets/filters.js`.

### The data & override files (this is how you change things)
| File | Purpose |
|---|---|
| `content/raw/products.json` | Original WooCommerce products |
| `content/raw/supplier_products.json` | Imported EDL dropship products |
| `content/raw/fridge_products.json` | Fridges rebuilt from stock take |
| `content/price_overrides.csv` | `product_id,price` — EME's own prices (wins over source) |
| `content/removed_products.csv` | `product_id` — products dropped from the site |
| `content/category_overrides.csv` | `product_id,category_id` — recategorise a product |
| `content/category_images.csv` | `slug,image` — custom category hero/card image |
| `content/image_map.json` | url→local path (2,300+ images localised into `assets/img/`) |

## ⚠️ Deployment — READ THIS
Live site is GitHub Pages served from a **`/EME/` subpath**, so all internal links must be prefixed with `/EME/`. The build does this when `BASE_PATH=/EME` is set.

**To publish changes:**
```
BASE_PATH=/EME python3 tools/build.py
git add -A && git commit -m "..." && git push origin main
```
Pages redeploys in ~60s. **Do NOT commit a build made without `BASE_PATH=/EME` to `main`** — it re-breaks the CSS/images (links point to the domain root instead of `/EME/`).

- **Local preview:** `python3 tools/build.py` (no BASE_PATH) → serves correctly at root (localhost). But don't commit that to main.
- **Permanent fix (recommended):** point a custom domain (e.g. eddiemaguire.ie) at Pages → served from root → drop `BASE_PATH` entirely and this whole issue disappears.

## The proven per-department workflow (only FRIDGES done so far)
To make a category accurate to real stock:
1. **Clear it** → add its product IDs to `removed_products.csv`.
2. **Stock take** — David prints the EasyRetail "Stock Take List" for that Main Group; transcribe to a CSV (model, type, qty). That = what's actually in stock.
3. **Photos** — David pulls official product images from the **brand dealer portals** into a folder; match by model.
4. **Prices** — generate a `model,price` sheet, David fills it, apply to `price_overrides.csv`.
5. Rebuild → the category is a true match to the shop floor.
